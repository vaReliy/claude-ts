# CTS payload manifest — paths copied into target projects by cts-sync.sh.
# One path per line. Trailing "/" marks a directory (copied recursively).
# Lines starting with "#" and blank lines are ignored.
#
# Ownership model (two-layer distribution): every payload path here is
# CTS-owned and plain-overwritten on sync — no merging, no baselines. Consumer
# customizations live in a SEPARATE, never-payload'd path instead: rules/cts/**
# pairs with rules/local/**, .claude/agents/*.md pairs with
# .claude/agents-local/*.md, AGENTS.md pairs with AGENTS.local.md, CLAUDE.md
# pairs with CLAUDE.local.md, and .cts/settings.cts.json layers under the
# consumer's own .claude/settings.json via jq deep-merge (see cts-sync.sh).
# None of the "local"/consumer-owned paths are listed below, and cts-sync.sh
# only ever touches paths listed here — so they are never synced, by
# construction, not by a special-case guard.
