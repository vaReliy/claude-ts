# Knowledge Inbox

Append-only queue for durable, project-relevant learnings whose final home isn't clear yet. Distilled into PROJECT_CONTEXT.md / CLAUDE.md / a rule / a skill, then deleted from here — this file should trend toward empty.

In the claude-ts host repo itself, stage an entry here only when the destination is genuinely uncertain — when it's already known (a specific line in CLAUDE.md/AGENTS.md/a rule/a skill), distill directly there instead. See CLAUDE.md's Knowledge capture section.
